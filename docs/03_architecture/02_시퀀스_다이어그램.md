# LessonPack AI 시퀀스 다이어그램

## 1. 문서 목적

이 문서는 현재 프론트엔드와 FastAPI 백엔드가 프로젝트 생성, 자료 적재, RAG 패키지 생성, 자연어 재생성 및 다운로드를 처리하는 순서를 정의한다.

## 2. 프로젝트 생성·자료 적재·패키지 생성

```mermaid
sequenceDiagram
    autonumber
    actor User as 사용자
    participant FE as Lovable React UI
    participant API as FastAPI
    participant Parser as Parser·Chunk Service
    participant Embed as LiteLLM Embedding
    participant DB as Supabase PostgreSQL·pgvector
    participant Storage as Supabase Storage
    participant LLM as LiteLLM
    participant OpenAI as OpenAI
    participant Gemini as Gemini
    participant Trace as Langfuse

    User->>FE: 강의 유형·대상·시간·차시·질의 입력
    FE->>API: POST /api/projects
    API->>DB: 프로젝트 메타데이터 저장
    DB-->>API: project_id
    API-->>FE: Project

    opt 교재 업로드
        User->>FE: PDF·DOCX·PPTX·TXT·MD 업로드
        FE->>API: POST /api/projects/{id}/materials
        API->>Parser: 텍스트 추출·정규화·청킹
        Parser-->>API: chunk와 출처 메타데이터
        API->>Embed: chunk 임베딩 요청
        Embed->>OpenAI: text-embedding-3-small
        OpenAI-->>Embed: embedding_v2
        API->>DB: document·chunk·embedding upsert
        DB-->>API: 적재 결과
        API-->>FE: MaterialIngestResult
    end

    opt 사용자 PPT 템플릿 업로드
        User->>FE: PPTX 템플릿 선택
        FE->>API: POST /api/projects/{id}/ppt-template
        API->>API: ZIP 안전성·레이아웃 분석
        API->>Storage: 원본 PPTX 저장
        API->>DB: 템플릿 메타데이터·매핑 저장
        API-->>FE: PPTTemplateMetadata
    end

    User->>FE: 패키지 생성 실행
    FE->>API: POST /api/projects/{id}/rag/generate<br/>queries[]
    loop 각 검색 질의
        API->>Embed: query 임베딩
        Embed->>OpenAI: embedding 요청
        OpenAI-->>Embed: query vector
        API->>DB: match_lessonpack_chunks_v2<br/>프로젝트 자료 우선 + baseline
        DB-->>API: 근거 chunk 후보
    end
    API->>DB: retrieval run·선정 근거 저장
    API->>LLM: 프로젝트 조건 + 근거 + 구조화 출력 스키마
    LLM->>OpenAI: primary completion
    alt OpenAI 실패 또는 제한
        LLM->>Gemini: fallback completion
        Gemini-->>LLM: JSON 응답
    else OpenAI 성공
        OpenAI-->>LLM: JSON 응답
    end
    LLM->>Trace: OTEL trace·모델·토큰·비용·지연
    LLM-->>API: 교안·실습·평가 패키지
    API->>DB: generation run 메타데이터 저장
    API-->>FE: package + retrieval_run_id + trace_id
    FE-->>User: 패키지 결과와 NCS 커버리지 표시
```

## 3. 자연어 재생성과 다운로드

```mermaid
sequenceDiagram
    autonumber
    actor User as 사용자
    participant FE as Lovable React UI
    participant API as FastAPI
    participant DB as Supabase pgvector
    participant LLM as LiteLLM
    participant Trace as Langfuse
    participant Storage as Supabase Storage
    participant Export as Export Service

    User->>FE: 자연어 수정 지시 입력
    FE->>API: POST /api/packages/{package_id}/regenerate
    API->>API: 원본 패키지·프로젝트 확인
    API->>DB: 수정 지시를 query로 근거 검색
    DB-->>API: 추가 근거 chunk
    API->>API: 원본 인용 근거 + 신규 근거 병합
    API->>LLM: 원본 패키지 + 수정 지시 + 병합 근거
    LLM->>Trace: 재생성 trace·입출력 정책·사용량
    LLM-->>API: 수정된 구조화 패키지
    API->>DB: 새 generation run 저장
    API-->>FE: 새 package_id와 수정 패키지
    FE->>FE: 현재 패키지를 응답 패키지로 교체
    FE-->>User: 변경된 결과 표시

    alt DOCX 다운로드
        User->>FE: DOCX 다운로드
        FE->>API: GET /api/packages/{new_id}/export.docx
        API->>Export: DOCX 생성
        Export-->>API: 임시 파일
        API-->>FE: Content-Disposition 첨부 응답
    else PPTX 다운로드
        User->>FE: PPTX 다운로드
        FE->>API: GET /api/packages/{new_id}/export.pptx
        API->>DB: 템플릿 메타데이터 조회
        API->>Storage: 사용자 PPTX 다운로드
        alt 사용자 템플릿 정상
            API->>Export: 사용자 레이아웃 기반 PPTX 생성
        else 템플릿 없음 또는 처리 실패
            API->>Export: 기본 PPTX 생성
        end
        Export-->>API: 임시 파일과 template mode
        API-->>FE: PPTX 첨부 응답
    end
    FE-->>User: 브라우저 파일 저장
```

## 4. NCS 공식 API 동기화

```mermaid
sequenceDiagram
    autonumber
    participant GH as GitHub Actions ncs-sync
    participant Sync as NCS Sync Script
    participant NCS as data.go.kr·NCS API
    participant Build as NCS Normalizer·Chunk Builder
    participant Embed as LiteLLM Embedding
    participant DB as Supabase

    GH->>Sync: 스케줄 또는 수동 실행
    Sync->>NCS: 능력단위 카탈로그 조회
    NCS-->>Sync: 코드·명칭·분류
    Sync->>DB: 카탈로그 upsert

    loop 상세 백필 대상
        Sync->>NCS: 수행준거·학습모듈 상세 조회
        alt 상세 응답 존재
            NCS-->>Sync: 상세 데이터
            Sync->>Build: 정규화·출처 메타데이터·chunk 생성
            Build->>Embed: chunk 임베딩
            Embed-->>Build: embedding_v2
            Build->>DB: lessonpack_chunks upsert
            Sync->>DB: 상세 적재 상태 갱신
        else 상세 응답 없음
            NCS-->>Sync: 빈 결과 또는 미지원
            Sync->>DB: 수행준거 0개 상태 유지·오류 기록
        end
    end
    Sync-->>GH: 조회·성공·실패·스킵 건수
```

## 5. 예외 계약

| 조건 | API 결과 | 사용자 동작 |
| --- | --- | --- |
| 검색 근거 없음 | `422` | 자료를 추가하거나 질의를 구체화 |
| LLM 생성 실패 | `502` | 잠시 후 재시도하며 Langfuse trace 확인 |
| Supabase 또는 저장소 장애 | `503` | health와 외부 서비스 상태 확인 |
| 프로젝트·패키지 없음 | `404` | 현재 프로젝트 또는 최신 package ID 재확인 |
| NCS 상세 근거 부족 | `422` 또는 수행준거 0개 표시 | 사용자 교재 업로드 또는 상세 동기화 수행 |
| 사용자 PPT 템플릿 처리 실패 | 기본 PPTX fallback | 응답 헤더의 template mode 확인 |

## 6. 현재 영속성 경계

- 프로젝트, 문서, chunk, retrieval run, generation run 메타데이터와 PPT 템플릿은 Supabase에 저장된다.
- 생성된 패키지 본문과 generation log 조회 객체는 현재 FastAPI 프로세스 메모리에 저장된다.
- DOCX/PPTX는 요청 시 임시 디렉터리에 생성된다.
- 따라서 컨테이너 재시작 이후 과거 package ID 재조회가 필요한 운영 요구는 별도 패키지 본문 영속화가 필요하다.
