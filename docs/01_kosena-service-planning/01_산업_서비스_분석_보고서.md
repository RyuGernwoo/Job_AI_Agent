## 1. 산업군 선정

| 항목 | 내용 |
|---|---|
| 산업군 | AI 기반 직업훈련 콘텐츠 제작 보조 서비스 |
| 인접 산업 | EdTech, HRD, LMS, 생성형 AI SaaS, 디지털 교재 제작 도구 |
| 핵심 고객 | 직업훈련 강사, 교육기관 운영 담당자 |
| 최종 수혜자 | 직업훈련 수강생 |
| 해결 문제 | 교안·실습·평가 문항 반복 작성, NCS·교재 근거 추적, 산출물 품질 편차 |

이 프로젝트는 “AI가 강사를 대체하는 서비스”가 아니라, **강사가 검수 가능한 초안을 빠르게 만들고 근거를 확인하는 보조 도구**로 정의한다.

## 2. 대표 서비스 분석

| 서비스 | 주요 기능 | 시사점 |
|---|---|---|
| Easygenerator EasyAI | 문서 기반 과정 초안 생성, 질문·학습 요소 생성, 지식 베이스 활용 | 교재 업로드 기반 강의 초안 생성은 실제 시장에서 채택된 기능 |
| Coursebox AI | AI course creator, quiz, flashcard, scenario, AI grading, LMS/SCORM/LTI 연동 | 평가·실습·과정 생성은 수요가 있으나 LMS 연동은 MVP 범위 초과 |
| MagicSchool AI | 수업 계획, 프레젠테이션, 루브릭, 객관식 퀴즈, 피드백 생성 | 교사용 업무별 AI 도구 묶음이 강한 제품 방향 |
| Articulate Rise | 웹 기반 과정 제작, 인터랙티브 콘텐츠, AI Assistant, LMS 게시 | 결과물 UX와 과정 제작 흐름 참고 |
| Dify / RAGFlow | 지식 베이스 기반 RAG 앱 구성, 문서 검색, 근거 기반 응답 | MVP의 RAG 구조 설계 근거 |

## 3. PESTEL 분석

| 영역 | 분석 | 기회/위협 |
|---|---|---|
| Political | 정부와 공공기관은 디지털 역량, AI, 직업능력개발을 강조한다. 직업훈련 과정은 NCS 기반 문서화가 중요하다. | 공공·직업훈련 수요는 기회, 공공 기준 미충족은 위협 |
| Economic | 강사 인건비와 콘텐츠 제작 시간이 교육기관 비용으로 작용한다. | 반복 문서 작성 자동화는 비용 절감 기회 |
| Social | 성인 학습자와 재직자는 짧은 기간 안에 실무형 학습 결과를 원한다. | 실습·평가 중심 콘텐츠 수요 증가 |
| Technological | LLM, RAG, 문서 파싱, 자동 문서 생성 기술이 성숙하고 있다. | 개인 MVP 구현 가능성이 높아짐 |
| Environmental | 디지털 문서와 템플릿 재사용으로 종이 출력과 중복 작성 부담을 줄일 수 있다. | 보조 가치 |
| Legal | 교재 저작권, 개인정보보호, 생성물 출처 표기가 중요하다. | 출처 표시와 강사 검수는 필수 |

## 4. Porter Five Forces

| 요인 | 강도 | 근거 |
|---|---|---|
| 기존 경쟁자 | 높음 | 이미 AI 교육 콘텐츠 생성 서비스가 존재한다. |
| 신규 진입 위협 | 높음 | LLM API와 RAG 프레임워크로 빠르게 MVP를 만들 수 있다. |
| 대체재 위협 | 높음 | ChatGPT, CLOVA Studio, Google Docs, PPT 템플릿으로도 대체 가능하다. |
| 구매자 교섭력 | 중간~높음 | 교육기관은 가격 민감도가 높고 기존 도구와 비교한다. |
| 공급자 교섭력 | 중간 | LLM API, embedding API, 클라우드 인프라에 의존한다. |

## 5. SWOT 분석

| 구분 | 내용 |
|---|---|
| Strengths | NCS·커리큘럼·교재를 함께 입력받는 직업훈련 특화 구조, HITL 검수 구조 |
| Weaknesses | 개인 프로젝트라 데이터 규모와 UI 완성도 제한, LMS 연동 경험 부족 |
| Opportunities | AI 교육 과정 증가, 강사 업무 자동화 수요, RAG 기반 출처 표시 요구 |
| Threats | 상용 서비스의 빠른 확장, 저작권 이슈, 생성 결과 품질 편차 |

## 6. Value Chain 분석

| 단계 | 기존 문제 | 서비스 개입 |
|---|---|---|
| 콘텐츠 기획 | 차시별 구조를 매번 새로 작성 | 커리큘럼 기반 차시 구조 제안 |
| 자료 탐색 | 교재에서 관련 내용을 직접 탐색 | RAG로 관련 문단 검색 |
| 산출물 작성 | 교안·실습·평가를 따로 작성 | 강의 패키지 단위 생성 |
| 품질 검수 | 강사별 형식과 난이도 차이 | 템플릿·루브릭·출처 표시 |
| 운영 문서화 | DOCX/PPTX 정리 시간이 추가됨 | 승인 결과 자동 export |

## 7. KSF 5개

| KSF | 설명 |
|---|---|
| 근거 기반 생성 | 교재·NCS 근거가 없으면 신뢰하기 어렵다. |
| 강사 검수 편의성 | AI 초안을 빠르게 고칠 수 있어야 한다. |
| 산출물 형식 표준화 | 교육기관 제출 문서로 바로 쓸 수 있어야 한다. |
| 1개월 구현 가능성 | 개인 프로젝트이므로 1개 차시 end-to-end에 집중한다. |
| 저작권·출처 관리 | 교재 기반 생성에는 출처와 사용 범위 관리가 필요하다. |

## 8. HMW 질문과 서비스 방향

| HMW 질문 | 서비스 방향 |
|---|---|
| 어떻게 하면 강사가 교재를 다시 읽지 않고 핵심 내용을 빠르게 찾을 수 있을까? | RAG 기반 관련 문단 검색 |
| 어떻게 하면 교안·실습·평가를 한 번에 묶을 수 있을까? | 1개 차시 강의 패키지 생성 |
| 어떻게 하면 AI 결과를 안전하게 검수할 수 있을까? | 강사 검수·승인 상태 모델 |
| 어떻게 하면 NCS와 산출물 연결을 보여줄 수 있을까? | NCS 태그와 citation ID 표시 |
| 어떻게 하면 개인 프로젝트로도 상용 서비스처럼 보일 수 있을까? | 업로드→생성→검수→다운로드 흐름 집중 |

## 9. 결론

직업훈련 강의 운영 보조 AI Agent는 이미 존재하는 범용 교육 콘텐츠 생성 서비스와 경쟁해야 한다. 따라서 넓은 LMS 플랫폼을 만들기보다, **한국 직업훈련 강사의 NCS·교재 기반 산출물 작성 문제**에 좁게 집중하는 것이 타당하다.

MVP는 “1개 차시의 교안·실습·평가 초안을 근거 기반으로 생성하고, 강사가 검수 후 DOCX로 내려받는 서비스”로 정의한다.
