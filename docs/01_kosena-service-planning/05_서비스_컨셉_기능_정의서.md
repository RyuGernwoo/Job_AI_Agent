# KOSENA 05. 서비스 컨셉·기능 정의서

프로젝트명: **LessonPack AI MVP**
목적: 고객 문제와 가치 제안을 연결하고, MVP에서 구현할 핵심 기능과 Use Case를 정의한다.

---

## 1. 서비스 컨셉

**LessonPack AI MVP**는 직업훈련 강사가 커리큘럼, NCS 능력단위, 교재 자료를 입력하면 1개 차시의 교안·실습·평가 패키지를 생성하고, 자연어로 조정한 뒤 DOCX/PPTX로 내려받을 수 있게 하는 RAG 기반 강의 운영 보조 Agent다.

핵심 컨셉:

- 교재와 NCS를 근거로 생성한다.
- AI 결과는 자연어 수정 요청으로 새 버전을 만들 수 있다.
- 최종 산출물은 운영 문서로 사용할 수 있게 export한다.

## 4. 기능 상세

### F-01. 교재 업로드·텍스트 추출

- 입력: PDF/DOCX/TXT 중 MVP에서 지원 가능한 형식
- 출력: 문서명, chunk ID, 원문 텍스트
- 주의: 저작권이 확인된 샘플 자료만 사용

### F-02. RAG 근거 검색

- 입력: 차시명, 학습목표, NCS 능력단위
- 처리: 관련 문단 3~5개 검색
- 출력: 문서명, chunk ID, 관련 문단

### F-03. 교안 초안 생성

- 구성: 차시명, 학습목표, 준비물, 도입·전개·정리, 강사 메모
- 조건: 각 핵심 설명에 citation ID 연결

### F-04. 실습 시나리오 생성

- 구성: 실습 배경, 수행 절차, 제출물, 평가 기준
- 조건: 실습 수행 절차는 최소 3단계 이상

### F-05. 평가 문항 생성

- 구성: 객관식 5문항, 정답, 해설, 실습형 평가 1개
- 조건: 정답과 해설 누락 금지

### F-06. 자연어 패키지 수정

- 상태: generated → regenerated
- 기능: 기존 패키지와 자연어 지시를 이용한 새 버전 생성
- 목적: AI 결과의 책임을 강사가 최종 확인

### F-07. DOCX 다운로드

- 구성: 표지, 교안, 실습, 평가, 근거 출처, 생성 로그 요약
- 조건: generated/regenerated 상태에서 즉시 export

## 5. Use Case 3종

| Use Case | Actor | Trigger | Main Flow | Success |
|---|---|---|---|---|
| UC-01. 1개 차시 강의 패키지 생성 | 강사 | 새 차시 준비 필요 | 과정 정보 입력 → 교재 업로드 → 근거 검색 → 교안·실습·평가 생성 | 생성 결과에 citation ID 포함 |
| UC-02. 생성 결과 수정 | 강사/운영자 | AI 패키지 생성 완료 | 자연어 수정 요청 → 새 패키지 생성 → 원본 보존 | regenerated 상태와 새 ID 저장 |
| UC-03. 운영 문서 export | 강사/운영자 | 패키지 생성 완료 | DOCX 다운로드 → 필요 시 PPTX 요약 다운로드 → 로그 확인 | 파일 생성 성공 |

## 7. 비기능 요구사항

| 항목 | 요구사항 |
|---|---|
| 신뢰성 | 생성 결과 핵심 항목 90% 이상 citation 연결 |
| 재현성 | 동일 입력에서 동일한 JSON 필드 구조 유지 |
| 사용성 | 1개 차시 생성 흐름을 발표 중 시연 가능 |
| 보안 | 저작권 미확인 자료와 개인정보 포함 자료 사용 금지 |
| 확장성 | 향후 다중 차시 생성과 PPTX 안정화 가능 구조 |

## 8. 결론

서비스 컨셉의 핵심은 **근거 있는 패키지 생성과 자연어 기반 반복 개선**이다. 기능은 많아 보이지만 MVP에서는 1개 차시, 1개 교재, 1개 강의 패키지에 집중해 end-to-end 흐름을 완성한다.
