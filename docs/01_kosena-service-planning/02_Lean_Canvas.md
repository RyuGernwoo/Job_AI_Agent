# KOSENA 02. Lean Canvas

프로젝트명: **LectureOps Agent MVP**  
서비스 한 줄 정의: 교재와 NCS를 근거로 1개 차시의 교안·실습·평가 초안을 생성하고, 강사가 검수 후 문서로 내려받는 RAG 기반 강의 운영 보조 Agent.

---

## 2. Problem 상세

| 문제 | 현재 대안 | 현재 대안의 한계 |
|---|---|---|
| 교안·실습·평가 반복 작성 | 기존 파일 복사 후 수정 | 오래 걸리고 품질 편차가 큼 |
| NCS·교재 근거 정리 | 사람이 직접 문서 탐색 | 출처 추적이 어렵고 누락 가능 |
| 생성형 AI 활용의 불안정성 | ChatGPT/CLOVA Studio 직접 사용 | 근거 불명확, 형식 불일치, 검수 기록 부족 |

---

## 3. Customer Segments

### Early Adopter

- 생성형 AI 또는 데이터 분석 직업훈련 과정을 운영하는 강사
- 매 차시 교안·실습·평가 문서를 제출해야 하는 강사
- 강사 산출물 형식을 관리하는 소규모 교육기관 운영자

### 제외 고객

- 학습자용 LMS 전체 운영이 필요한 대형 기관
- 자동 채점·성적 관리가 핵심인 평가 플랫폼 사용자
- 자체 콘텐츠 제작팀과 LMS를 이미 보유한 대형 기업교육 조직

---

## 4. Unique Value Proposition

> **“교재와 NCS를 근거로 1개 차시 강의 패키지를 10분 안에 초안화하는 강사 보조 AI”**

가치 제안의 핵심은 속도보다 **근거 기반 생성 + 강사 검수 가능성**이다. 완전 자동화가 아니라, 강사가 책임지고 수정할 수 있는 초안을 빠르게 제공한다.

---

## 5. Solution

| 기능 | 문제 해결 방식 |
|---|---|
| 교재 업로드 | 강사가 가진 자료를 생성 근거로 사용 |
| RAG 검색 | 차시와 관련 있는 교재 문단 추출 |
| 교안 생성 | 도입·전개·정리 구조로 강의 초안 작성 |
| 실습 생성 | 시나리오, 수행 절차, 제출물, 루브릭 생성 |
| 평가 생성 | 객관식 5문항 + 실습형 평가 1개 생성 |
| HITL 검수 | 강사가 수정·승인 후 최종 확정 |
| DOCX export | 운영 문서로 제출 가능한 형태 제공 |

## 6. 핵심 가설 3개

| 가설 | 검증 방법 | 성공 기준 |
|---|---|---|
| H1. 강사는 교안·실습·평가 초안 작성 시간을 줄이는 데 높은 가치를 느낀다. | 강사 인터뷰 또는 사용성 테스트 | 수작업 대비 빠르다는 반응 확보 |
| H2. 근거 문단 표시가 있으면 AI 생성물 신뢰도가 올라간다. | citation 표시 버전과 미표시 버전 비교 | citation 표시 버전 선호 |
| H3. 1개 차시 생성만으로도 MVP 가치가 있다. | 1개 차시 데모 후 피드백 수집 | 다른 차시에도 쓰고 싶다는 반응 확보 |

## 8. 결론

Lean Canvas 기준으로 이 서비스의 초기 생존 전략은 넓은 LMS 시장 전체를 노리는 것이 아니다. **직업훈련 강사의 반복 산출물 작성 부담**이라는 좁고 구체적인 문제를 해결하고, 교재 근거와 강사 검수를 차별점으로 삼는 것이 가장 현실적이다.
