# KOSENA 07. 최종 발표자료 구성안

프로젝트명: **LessonPack AI MVP**
발표 시간: 5분
발표 형식: 15~20쪽 PPT/PDF로 확장 가능하되, 실제 5분 발표는 7장 내외 핵심 흐름으로 구성한다.

---

## 1. 발표 핵심 메시지

> 직업훈련 강사는 매 차시 교안·실습·평가를 반복 작성한다. LessonPack AI MVP는 교재와 NCS를 근거로 1개 차시 강의 패키지를 생성하고, 강사가 검수한 뒤 DOCX로 내려받게 하는 RAG 기반 강의 운영 보조 Agent다.

---

## 2. 5분 발표 흐름

| 슬라이드 | 제목 | 핵심 내용 | 시간 |
|---|---|---|---|
| 1 | 문제 정의 | 강사의 반복 산출물 작성 부담, NCS·교재 근거 추적 어려움 | 40초 |
| 2 | 고객과 Pain Point | 강사·운영자 페르소나, CJM 핵심 Pain Point | 40초 |
| 3 | 시장·경쟁 분석 | 상용 서비스와 오픈소스 대비 차별점 | 40초 |
| 4 | 서비스 컨셉 | RAG + HITL + DOCX export 구조 | 50초 |
| 5 | MVP 범위 | Must/Should/Won't Have, 1개 차시 집중 | 50초 |
| 6 | 데모 시나리오 | 업로드 → 생성 → 검수 → 다운로드 | 60초 |
| 7 | 로드맵과 한계 | 4주 개발 계획, MVP 한계, 추후 확장 | 40초 |

---

## 3. 슬라이드별 상세 구성

### Slide 1. 문제 정의

핵심 문장:

- 직업훈련 강사는 과정별 교안, 실습 과제, 평가 문항을 반복적으로 작성한다.
- NCS 능력단위와 교재 근거를 맞춰야 하지만 수작업으로는 시간이 많이 든다.
- 생성형 AI를 그냥 쓰면 근거와 출처가 불명확하다.

시각 자료:

- “교안 / 실습 / 평가 / 근거 출처”가 각각 흩어진 그림

### Slide 2. 고객과 Pain Point

핵심 문장:

- 강사는 빠른 초안과 수정 가능한 결과를 원한다.
- 운영자는 표준 형식과 근거 확인을 원한다.

표:

| 고객 | Pain Point | 원하는 결과 |
|---|---|---|
| 직업훈련 강사 | 반복 작성, 근거 추적 어려움 | 교안·실습·평가 초안 |
| 운영 담당자 | 문서 형식 편차, 품질 검수 부담 | 표준 템플릿, 승인 상태 |

### Slide 3. 시장·경쟁 분석

핵심 문장:

- Easygenerator, Coursebox, MagicSchool 등 AI 교육 콘텐츠 서비스는 이미 존재한다.
- 하지만 한국 직업훈련/NCS 문서 형식에 좁게 특화된 MVP는 차별화 가능하다.

비교 축:

- 범용 생성 도구 ↔ 직업훈련 특화
- 단순 텍스트 생성 ↔ 근거·검수·문서화 워크플로우

### Slide 4. 서비스 컨셉

핵심 문장:

- LessonPack AI는 교재를 검색하고, 근거 문단을 바탕으로 강의 패키지를 생성한다.
- AI 결과는 바로 확정되지 않고 강사가 검수한다.

아키텍처:

```text
입력 → 교재 업로드 → RAG 검색 → 생성 Agent → 강사 검수 → DOCX/PPTX export
```

### Slide 5. MVP 범위

Must Have:

- 커리큘럼/NCS 입력
- 교재 업로드
- RAG 근거 검색
- 교안·실습·평가 생성
- 강사 검수
- DOCX 다운로드

Won't Have:

- LMS 연동
- 자동 채점
- 기관 권한 관리
- 전체 과정 자동 생성

핵심 문장:

> 한 달 MVP는 1개 차시 end-to-end 흐름에 집중한다.

### Slide 6. 데모 시나리오

시연 순서:

1. 과정명, 차시명, NCS 능력단위 입력
2. 샘플 교재 업로드
3. 관련 문단 검색 결과 확인
4. 교안·실습·평가 초안 생성
5. 강사 검수·수정
6. DOCX 다운로드

성공 기준:

- 생성 결과에 citation ID가 포함된다.
- 강사가 내용을 수정할 수 있다.
- 승인된 결과가 DOCX로 저장된다.

### Slide 7. 로드맵과 한계

4주 로드맵:

| 주차 | 목표 |
|---|---|
| Week 1 | 샘플 데이터, 파일 파싱, RAG 검색 |
| Week 2 | 교안·실습·평가 생성 Agent |
| Week 3 | HITL 검수와 DOCX export |
| Week 4 | 품질 개선, 발표자료, 데모 안정화 |

한계:

- 전체 과정이 아니라 1개 차시 중심
- 자동 채점과 LMS 기능 제외
- NCS 공식 DB 자동 연동 제외
- 정량 시장 규모는 추가 1차 리서치 필요

---

## 4. 발표용 한 문장 요약

> LessonPack AI MVP는 직업훈련 강사가 교재와 NCS를 근거로 1개 차시의 교안·실습·평가 초안을 만들고, 검수 후 문서로 내려받게 하는 강의 운영 보조 AI 서비스입니다.

---

## 5. 예상 Q&A

| 질문 | 답변 |
|---|---|
| 왜 전체 과정 자동 생성이 아니라 1개 차시인가? | 한 달 개인 MVP에서는 품질 검증 가능한 범위가 중요하므로 1개 차시 end-to-end를 우선한다. |
| 기존 ChatGPT와 차이는 무엇인가? | ChatGPT는 범용 생성 도구이고, 이 서비스는 교재 근거 검색, citation, 강사 검수, DOCX export를 한 흐름으로 묶는다. |
| 저작권 문제는 어떻게 대응하는가? | 샘플 공개 자료 또는 사용 허가 자료만 사용하고, 생성 결과에 근거 문단을 표시한다. |
| 상용 서비스와 경쟁 가능한가? | 전체 기능 경쟁은 어렵다. 대신 한국 직업훈련/NCS 산출물 형식이라는 좁은 문제에 특화한다. |
| MVP 성공 기준은 무엇인가? | 업로드→생성→검수→DOCX 다운로드가 1개 샘플 과정에서 성공하는 것이다. |

## 7. 결론

발표의 중심은 “기능이 많다”가 아니라 “문제가 좁고, 해결 흐름이 끝까지 이어진다”이다. 5분 안에 문제, 고객, 차별점, 데모, 로드맵이 한 줄로 이어지면 충분히 설득력 있는 서비스 기획 발표가 된다.
