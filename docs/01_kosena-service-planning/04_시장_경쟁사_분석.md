# KOSENA 04. 시장·경쟁사 분석

프로젝트명: **LessonPack AI MVP**
목적: AI 기반 직업훈련 강의 운영 보조 서비스의 시장 범위, 경쟁 구도, 포지셔닝을 정의한다.

---

## 1. 시장 정의

| 구분 | 내용 |
|---|---|
| 전체 시장 | 직업훈련·성인교육 콘텐츠 제작 보조 시장 |
| 도달 가능 시장 | AI·디지털·프로그래밍 직업훈련 과정의 강의 산출물 제작 시장 |
| 초기 목표 시장 | 개인 강사와 소규모 교육기관의 1개 차시 산출물 자동 초안 생성 수요 |

이 프로젝트는 초기 단계에서 전체 LMS 시장을 겨냥하지 않는다. **직업훈련 강사의 수업 준비 산출물 작성**이라는 좁은 시장에서 실험한다.

---

## 2. TAM/SAM/SOM 산정 방식

현재 단계에서는 검증되지 않은 시장 규모 숫자를 임의로 제시하지 않는다. KOSENA 기준에 맞춰 산식과 데이터 수집 계획을 먼저 제시한다.

| 구분 | 정의 | 산식 |
|---|---|---|
| TAM | 국내 직업훈련·성인교육 콘텐츠 제작 보조 전체 잠재시장 | `직업훈련/성인교육 강사 수 × 연간 콘텐츠 제작 도구 지출액` |
| SAM | AI·디지털·프로그래밍 과정처럼 실습·평가 생성 수요가 큰 도달 가능 시장 | `AI/디지털 직업훈련 과정 수 × 과정당 강사/운영자 수 × 예상 지불액` |
| SOM | 1~3년 안에 개인·소규모 기관 대상으로 확보 가능한 초기 시장 | `초기 접촉 가능 기관 수 × 전환율 × 월/연 구독 단가` |

### 데이터 수집 후보

- 고용노동부·HRD-Net 공개 통계
- 직업훈련기관 수와 과정 수
- AI·디지털 과정 개설 수
- 강사 대상 설문 기반 지불 의향
- 유사 SaaS 가격대

## 3. 경쟁사 분류

| 분류 | 서비스 | 경쟁 이유 | 우리 서비스의 차별점 |
|---|---|---|---|
| 직접 경쟁 | Easygenerator, Coursebox AI, MagicSchool AI | AI 기반 교육 콘텐츠·퀴즈·과정 생성 기능 제공 | 한국 직업훈련/NCS 산출물 형식에 집중 |
| 간접 경쟁 | ChatGPT, CLOVA Studio, Google Docs/Slides | 강사가 직접 프롬프트로 교안·문항 생성 가능 | RAG 근거, 자연어 재생성, DOCX 템플릿을 한 흐름으로 제공 |
| 잠재 경쟁 | LMS 사업자, HRD 솔루션 기업 | 기존 교육 운영 시스템에 AI 기능을 붙일 수 있음 | 초기 MVP에서는 좁은 도메인과 빠른 실험으로 차별화 |
| 오픈소스 대안 | Dify, RAGFlow, Moodle, H5P | RAG·콘텐츠·문제은행 구성 가능 | 조립형 도구가 아니라 강사용 end-to-end 패키지 |

---

## 4. 경쟁 비교표

| 비교 항목 | LessonPack AI MVP | Easygenerator/Coursebox류 | ChatGPT/CLOVA 직접 사용 | Dify/RAGFlow 직접 구축 |
|---|---|---|---|---|
| 직업훈련 특화 | 높음 | 낮음~중간 | 낮음 | 낮음 |
| NCS 연결 | 수동 입력 기반 지원 | 제한적 | 프롬프트 의존 | 직접 설계 필요 |
| 근거 출처 표시 | MVP 핵심 | 서비스별 상이 | 사용자가 직접 요구해야 함 | 구현 가능 |
| 자연어 수정 | 핵심 흐름 | 서비스별 상이 | 새 대화로 수행 | 직접 설계 필요 |
| DOCX/PPTX 산출 | 핵심 산출물 | 제공 가능 | 별도 작업 필요 | 직접 구현 필요 |
| 구현 난이도 | 개인 MVP 가능 | 사용은 쉬우나 차별화 어려움 | 가장 쉬움 | 상대적으로 높음 |

## 5. 포지셔닝 맵

### 축 정의

| 축 | 낮음 | 높음 |
|---|---|---|
| X축 | 범용 생성 도구 | 직업훈련/NCS 특화 |
| Y축 | 단순 텍스트 생성 | 근거·수정·문서화 워크플로우 |

### 위치 해석

| 서비스군 | 위치 |
|---|---|
| ChatGPT/CLOVA 직접 사용 | 범용 생성 도구, 단순 텍스트 생성 |
| Dify/RAGFlow | 범용 RAG 구축 도구, 워크플로우는 직접 설계 필요 |
| Easygenerator/Coursebox | 교육 콘텐츠 제작 워크플로우는 강하지만 한국 직업훈련/NCS 특화는 낮음 |
| LessonPack AI MVP | 직업훈련 특화도 높음, 워크플로우 깊이는 MVP 수준 |

LessonPack AI MVP는 상용 서비스 전체 기능을 이기려는 전략이 아니다. **한국 직업훈련 강사의 차시별 산출물 작성**이라는 좁은 문제에서 명확한 포지션을 만든다.

---

## 6. 기회 요인

| 기회 | 설명 |
|---|---|
| 생성형 AI 교육 확산 | AI·디지털 과정이 늘수록 강의 산출물 작성 수요가 증가 |
| 강사 업무 부담 | 반복 산출물 작성은 자동화 가치가 높음 |
| RAG 기반 신뢰성 요구 | 출처가 있는 생성 결과에 대한 수요 증가 |
| 개인 MVP 가능성 | FastAPI, Dify, RAGFlow 등 참고 가능한 도구가 많음 |

---

## 7. 위협 요인

| 위협 | 대응 |
|---|---|
| 상용 서비스의 기능 확장 | 직업훈련/NCS 특화로 좁게 차별화 |
| 범용 LLM의 품질 향상 | 문서 export와 패키지 버전 워크플로우로 차별화 |
| 저작권 이슈 | 샘플 공개 자료 또는 사용 허가 자료만 사용 |
| 데이터 부족 | MVP는 1개 차시 샘플로 검증하고 리서치로 보완 |

---

## 8. 결론

시장 진입 전략은 “큰 교육 플랫폼”이 아니라 “직업훈련 강사용 산출물 생성 보조 도구”다. 경쟁은 강하지만, NCS·교재 근거·자연어 재생성·DOCX export를 묶은 좁은 MVP는 개인 프로젝트로 충분히 설득 가능한 포지션을 가진다.
