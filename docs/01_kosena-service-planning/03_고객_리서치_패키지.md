# KOSENA 03. 고객 리서치 패키지

프로젝트명: **LessonPack AI MVP**
목적: 직업훈련 강사와 교육 운영 담당자의 실제 Pain Point를 가설로 정의하고, MVP 검증을 위한 인터뷰·설문·사용성 테스트 계획을 수립한다.

---

## 1. 리서치 목표

| 목표 | 설명 |
|---|---|
| 고객 이해 | 강사와 운영자가 어떤 산출물 작성에 가장 큰 부담을 느끼는지 파악 |
| 문제 검증 | 교안·실습·평가 자동 초안 생성이 실제로 필요한지 확인 |
| 기능 우선순위 | RAG 근거 표시, DOCX 다운로드, HITL 검수 중 무엇이 중요한지 확인 |
| MVP 사용성 | 1개 차시 생성 흐름이 충분히 설득력 있는지 검증 |

## 4. Customer Journey Map

| 단계 | 사용자 행동 | 감정 | Pain Point | Opportunity |
|---|---|---|---|---|
| 인지 | 새 과정 또는 새 차시 운영 요청을 받음 | 부담 | 준비할 문서가 많음 | 과정 정보를 입력하면 필요한 산출물 목록 제안 |
| 고려 | 교재와 NCS 능력단위를 확인 | 막막함 | 어디서부터 교안을 짤지 모름 | 관련 교재 문단 자동 검색 |
| 사용 | AI로 교안·실습·평가 초안 생성 | 기대 | 생성 결과를 믿어도 되는지 확인 필요 | citation과 원문 비교 화면 |
| 검수 | 초안을 수정하고 승인 | 안도 | 형식 정리와 문서 변환 필요 | 표준 DOCX 템플릿 export |
| 재사용 | 다음 차시에도 활용 | 만족 또는 피로 | 설정을 매번 반복할 수 있음 | 과정 설정 복제, 템플릿 저장 |

## 5. Pain Point 정리

| Pain Point | 심각도 | 근거 가설 |
|---|---|---|
| 교안·실습·평가 반복 작성 | 높음 | 매 차시 반복되는 산출물이므로 시간 부담이 크다. |
| 근거 출처 추적 어려움 | 높음 | 교재 기반 수업에서는 출처와 정합성이 중요하다. |
| 산출물 형식 불일치 | 중간~높음 | 강사별 문서 구조가 달라 운영자가 취합하기 어렵다. |
| AI 생성 결과 신뢰 부족 | 높음 | 근거 없는 문장은 그대로 사용하기 어렵다. |
| 문서 export 번거로움 | 중간 | DOCX/PPTX 정리에 추가 시간이 든다. |

## 6. 1차 인터뷰 계획

| 항목 | 내용 |
|---|---|
| 대상 | 직업훈련 강사 3~5명, 교육 운영자 1~2명 |
| 방식 | 20~30분 반구조화 인터뷰 |
| 목적 | 반복 산출물 작성 부담, AI 사용 경험, 출처 표시 필요성 확인 |

### 질문 예시

1. 한 차시 교안·실습·평가를 준비하는 데 평균적으로 얼마나 걸리십니까?
2. 가장 작성하기 부담스러운 산출물은 무엇입니까?
3. 생성형 AI를 수업 준비에 사용해 본 적이 있습니까?
4. AI 결과를 그대로 쓰기 어려웠던 이유는 무엇입니까?
5. 교재 근거와 출처 표시가 있다면 신뢰도가 올라갈 것 같습니까?
6. DOCX/PPTX 자동 출력이 실제 운영에 도움이 되겠습니까?

## 8. 사용성 테스트 계획

| 항목 | 내용 |
|---|---|
| 대상 | 강사 또는 예비 강사 2~3명 |
| 시나리오 | 샘플 교재 업로드 → 1개 차시 생성 → 근거 확인 → 결과 수정 → DOCX 다운로드 |
| 관찰 지표 | 막히는 단계, 수정량, 근거 신뢰도, 재사용 의향 |

### 성공 기준

- 사용자가 1개 차시 생성 흐름을 이해한다.
- 생성 결과를 수정할 수 있다고 느낀다.
- citation이 신뢰도 향상에 도움이 된다고 응답한다.
- 다른 차시에도 써보고 싶다는 반응을 보인다.

---

## 9. 리서치 한계

- 현재 문서는 실제 인터뷰·설문 결과가 아니라 리서치 설계안이다.
- 정량 주장은 조사 후 갱신해야 한다.
- 페르소나는 가설이며 실제 고객 인터뷰를 통해 수정될 수 있다.

---

## 10. 결론

초기 고객은 “AI를 이미 조금 써봤지만, 결과를 수업 문서로 정리하는 데 여전히 시간이 드는 강사”로 잡는 것이 적절하다. MVP 검증은 기능 수보다 **강사가 실제로 수정·승인·다운로드까지 해볼 수 있는지**에 집중한다.
