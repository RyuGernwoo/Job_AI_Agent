# NCS 전체 카탈로그 검색 구축 및 운영 기준

## 1. 목적

LessonPack AI의 NCS 검색은 보유 RAG 데이터 범위와 무관하게 공식 능력단위 코드와 능력단위명을 검색할 수 있어야 한다. 상세 수행준거가 아직 전처리되지 않은 능력단위도 검색 결과에는 포함하되, 수행준거 수를 `0개`로 표시한다.

## 2. 공식 근거

- NCS 공식 사이트는 NCS 분류를 대분류 24개, 중분류 81개, 소분류 270개, 세분류 1,096개로 제시한다.
- NCS 공식 사이트의 검색 화면은 분류명, 능력단위명, 능력단위요소명 및 능력단위 코드 검색을 제공한다.
- 한국산업인력공단의 공공데이터포털 `NCS 기준정보 조회` API는 능력단위분류코드 조회(`NCS005`)와 능력단위 키워드 검색(`NCS007`)을 제공한다.
- 한국산업인력공단의 `국가직무능력표준 정보_20251231` CSV는 능력단위 코드, 명칭, 수준, 훈련시간 13,442건을 제공한다.

공식 원본:

- NCS 검색: https://www.ncs.go.kr/unity/th03/selectKeywordSearch.do
- NCS 기준정보 API: https://www.data.go.kr/data/15128213/openapi.do
- NCS 전체 능력단위 CSV: https://www.data.go.kr/data/15083321/fileData.do

## 3. 데이터 계층

| 계층 | 데이터 | 검색 노출 | 수행준거 표시 | 활용 |
|---|---|---:|---:|---|
| 공식 기준 카탈로그 | 능력단위 코드, 명칭, 수준, 버전 | 전체 | 0개 가능 | 코드·명칭 검색 및 식별 |
| LessonPack 상세 카탈로그 | 전처리된 NCS 문서와 수행준거 | 전체 카탈로그에 병합 | 1개 이상 | 수행준거 선택 및 NCS 정렬 |
| 프로젝트 업로드 RAG | 사용자가 업로드한 학습모듈·교재 | 검색 근거에 포함 | 사용자가 직접 입력 | 상세 데이터 미적재 분야의 생성 근거 |

`criteria=[]`는 공식 능력단위가 존재하지 않는다는 뜻이 아니라, LessonPack의 상세 수행준거 RAG가 아직 적재되지 않았다는 뜻이다.

## 4. 구축 절차

1. 공공데이터포털에서 최신 `한국산업인력공단_국가직무능력표준 정보` CSV를 내려받는다.
2. 기존 PDF/XLS 변환 결과에서 상세 능력단위와 수행준거를 생성한다.
3. 공식 CSV를 기준으로 상세 데이터를 능력단위 코드별 병합한다.
4. `lessonpack_ncs_catalog`에는 전체 능력단위를, `lessonpack_ncs_criteria`에는 확보된 수행준거만 적재한다.
5. `007_ncs_catalog_search.sql`을 적용해 코드·명칭 부분검색 인덱스를 생성한다.

```powershell
python scripts/prepare_ncs_catalog.py `
  --official-csv "C:\path\to\한국산업인력공단_국가직무능력표준 정보_20251231.csv" `
  --upload
```

## 5. 서비스 동작

- 검색어는 능력단위 코드 또는 능력단위명에 부분 일치한다.
- 정확한 코드 일치, 정확한 명칭 일치, 접두어 일치, 부분 일치 순으로 결과를 정렬한다.
- 상세 RAG가 있는 능력단위는 수행준거 목록을 선택할 수 있다.
- 상세 RAG가 없는 능력단위는 `수행준거 0개`, `RAG 미적재`로 표시한다.
- RAG 미적재 능력단위를 선택하면 공식 코드와 명칭은 고정하고, 사용자가 업로드할 NCS 자료를 근거로 수행준거를 직접 입력한다.
- 백엔드는 이 경우 `source_status=needs_review`로 저장해 상세 공식 수행준거 검증이 완료된 상태와 구분한다.

## 6. 검증 기준

- 공식 CSV 행 수와 파싱된 능력단위 수가 일치한다.
- 코드와 명칭 검색이 각각 동작한다.
- 정확한 코드 검색 결과가 첫 번째로 반환된다.
- 상세 데이터 미적재 항목의 `criteria`는 빈 배열이다.
- 기존 상세 능력단위의 수행준거는 공식 CSV 병합 후에도 유지된다.
- 상세 미적재 능력단위로 프로젝트를 만들 때 공식 코드·명칭은 유지되고 사용자 수행준거 입력이 허용된다.

## 7. 갱신 주기

공식 CSV는 연 1회 이상, NCS 기준정보 API 및 NCS 포털 변경 공지 확인 후 갱신한다. 새 공식 목록을 먼저 적재한 뒤 기존 상세 수행준거를 병합해야 하며, 공식 목록 적재만으로 기존 수행준거를 빈 배열로 덮어쓰면 안 된다.
