## 1. 목적

이 문서는 `data/` 폴더에 준비된 원천 데이터셋을 LessonPack AI MVP에서 어떻게 활용하고, 어떤 전처리 산출물로 변환할지 정의한다.

MVP의 데이터 처리 목표는 전체 자료를 모두 학습하거나 색인하는 것이 아니다. 한 달 MVP 범위에서는 **1개 차시 강의 패키지 생성에 필요한 근거 문단만 선별하고**, 검색·생성·검증에 쓸 수 있는 작은 데이터셋으로 정제한다.

### 2.1 NCS PDF to Markdown 변환 결과

NCS 폴더 분석은 PDF 원문이 아니라 변환된 Markdown 파일을 기준으로 진행한다. 2026-07-16 기준 `data/raw/materials/ncs/` 상위 폴더의 PDF 27개를 같은 파일명 `.md` sidecar로 변환했다.

변환 방식:

- 도구: PyMuPDF text extraction
- 출력: PDF와 같은 경로의 `.md` 파일
- 구조: 파일명 제목, 원본 PDF명, page count, 변환 메모, `## Page N` 단위 텍스트
- 주의: 표, 도표, 이미지, 페이지 레이아웃은 자동 추출만으로 완전하지 않으므로 최종 YAML 작성 시 수동 검토가 필요하다.

확인된 NCS 계열:

| 구분 | 값 |
|---|---|
| 대분류 | 20 정보통신 |
| 중분류 | 01 정보기술 |
| 소분류 | 02 정보기술개발 |
| 세분류 | 02 응용SW엔지니어링 |
| 능력단위 범위 | `LM2001020201` ~ `LM2001020235` 중 27개 학습모듈 |

MVP 우선 검토 후보:

| 우선순위 | MD 파일 | 능력단위 | 확인된 학습 내용 | 활용 판단 |
|---:|---|---|---|---|
| 1 | `LM2001020231_23v5_.프로그래밍+언어+활용_20251108.md` | 프로그래밍 언어 활용 | 구조적 프로그래밍 언어, 객체지향 프로그래밍 언어, 스크립트 언어 활용 | Python 기초 실습과 직접 연결 |

| 3 | `LM2001020235_23v1_자료구조+활용_20251108.md` | 자료구조 활용 | 기본 자료구조, 정렬 및 탐색 알고리즘 | Python list/dict 자료와 연결 |
| 4 | `LM2001020232_23v5_.응용SW+기초+기술+활용_20251108.md` | 응용SW 기초기술 활용 | 네트워크, 미들웨어, 데이터베이스 기초 | 확장 차시 또는 보조 NCS 후보 |

보류 후보:

- `요구사항 확인`, `애플리케이션 설계`, `화면 구현`, `테스트 수행`, `배포`, `패키징` 등은 소프트웨어 개발 전 과정에는 중요하지만, 현재 MVP의 1차시 주제인 Python 함수·자료구조·프롬프트 자동화와 직접 연결성이 낮다.
- 해당 모듈은 전체 과정 확장 또는 고급 차시 기획 단계에서 다시 검토한다.

## 3. 데이터셋별 활용 방안

### 3.1 Python 공식 문서

Python 문서는 MVP의 핵심 교재 원문으로 사용한다.

우선 활용 범위:

| 파일 | 활용 목적 | 처리 우선순위 |
|---|---|---:|
| `tutorial_functions.md` | 함수 정의, 매개변수, 반환값, docstring 설명 | 1 |
| `python_tutorial_data_structures.md` | list, stack, queue, dictionary 등 기초 자료구조 설명 | 1 |
| `python-3.14-docs-text/tutorial/controlflow.txt` | 원본 검증 및 추가 chunk 후보 | 2 |
| `python-3.14-docs-text/tutorial/datastructures.txt` | 원본 검증 및 추가 chunk 후보 | 2 |
| `python-3.14-docs-text/tutorial/inputoutput.txt` | 입출력 실습 확장 | 3 |
| `python-3.14-docs-text/tutorial/modules.txt` | 모듈화 실습 확장 | 3 |

전처리 원칙:

- 문서 전체를 무작정 chunking하지 않는다.
- `4.8 Defining Functions`, `5.1 More on Lists`, `5.5 Dictionaries`처럼 수업 목표와 직접 연결되는 섹션만 사용한다.
- 원문 chunk마다 `source_name`, `source_url`, `license`, `section`, `source_file`, `chunk_id`를 부여한다.
- 코드 예제와 설명 문단은 같은 chunk에 들어가도록 우선 보존한다.

### 3.2 pandas 10분 튜토리얼

pandas 자료는 현재 MVP 주제인 “Python 함수와 프롬프트 자동화 실습”과 직접 연결성은 Python 튜토리얼보다 낮다. 다만 직업훈련 과정명에 “데이터 분석” 맥락이 포함될 경우 확장 차시로 활용할 수 있다.

활용 방안:

- MVP 1차 검증에서는 선택 데이터로 둔다.
- `Series`, `DataFrame`, 객체 생성, 데이터 확인 섹션만 추출한다.
- RST 지시자, `.. ipython::`, `:class:` 같은 Sphinx 문법은 평문으로 정리한다.
- 코드 블록은 실습 생성에 유용하므로 제거하지 않는다.

### 3.3 고용24 훈련과정 metadata

`employ24_metadata.md`는 실제 훈련과정의 시장 맥락과 과정명 표현을 참고하기 위한 데이터다.

활용 범위:

- 서비스 기획서와 발표자료의 시장 예시 보강
- 샘플 커리큘럼의 과정명·학습 대상 문장 개선
- 생성 프롬프트의 `training_context` 보조 입력

제한:

- RAG 근거 문단으로 직접 사용하지 않는다.
- 교육 내용 생성의 사실 근거로 사용하지 않는다.
- 출처 URL, 조회일, 수동 정리 여부를 `README_DATASET.md`에 명시해야 한다.

### 3.4 NCS 자료

NCS 자료는 LessonPack AI의 직업훈련 정합성을 만드는 핵심 데이터다. 업데이트된 `ncs` 폴더에는 응용SW엔지니어링 계열 NCS 학습모듈 PDF가 충분히 준비되어 있으므로, 이전처럼 “NCS 원천 부족” 상태는 아니다. 다만 서비스 입력과 평가에 쓰려면 PDF/MD 전체가 아니라 **선별된 NCS 요약 YAML**이 필요하다.

활용 원칙:

- 분석과 전처리는 PDF가 아니라 변환된 `.md` 파일 기준으로 수행한다.
- 전체 27개 NCS 모듈을 모두 RAG에 넣지 않는다.
- MVP 1차 범위는 `프로그래밍 언어 활용`, `프로그래밍 언어 응용`, `자료구조 활용` 3개를 중심으로 좁힌다.
- NCS 학습모듈의 “활용 시 유의 사항”에 따라 출처를 명시하고 교육적 목적 사용 범위를 기록한다.
- 도표·사진·삽화·도면 등 제3자 저작물이 포함될 수 있으므로, 원문 전체 재배포나 이미지 추출 사용은 피한다.

MVP 활용 방안:

| 용도 | 사용할 NCS 자료 | 처리 방식 |
|---|---|---|
| 커리큘럼 정합성 | `프로그래밍 언어 활용`, `프로그래밍 언어 응용` | YAML의 `performance_criteria`, `knowledge`, `skills`로 요약 |
| 실습 설계 | `자료구조 활용`, `프로그래밍 언어 활용` | 실습 주제 후보와 평가 기준 추출 |
| 평가 문항 생성 | `프로그래밍 언어 응용`, `자료구조 활용` | NCS 연결 문장과 루브릭 작성 |
| RAG 근거 | NCS MD 중 선별 section | 원문 chunk는 최소 범위만 사용, citation 포함 |
| 발표/기획 근거 | NCS 모듈명과 능력단위 코드 | 서비스의 직업훈련 특화 근거로 사용 |

필수 보완:

- `data/raw/materials/ncs/ncs.yaml`을 0바이트 상태로 두지 않고, MD 분석 기반의 `data/raw/ncs/ncs_application_sw_programming.yaml`로 재작성한다.
- Excel 보고서는 `data/raw/ncs/reports/`로 정리하고, 실제 YAML 작성에는 우선 사용하지 않는다.
- 변환된 MD는 `data/raw/ncs/converted_md/`로 정리하거나, 현재 위치를 유지한다면 `selected_sources.yaml`에 명시적으로 경로를 고정한다.

## 4. 목표 데이터 디렉터리 구조

현재 `data/raw/materials/`에 원천 데이터가 몰려 있으므로, 전처리 전후 구조를 아래처럼 분리한다.

```text
data/
  raw/
    curriculum/
      curriculum_python_prompt_automation.yaml
    ncs/
      source_pdf/
        LM2001020231_23v5_programming_language_use.pdf
        LM2001020230_23v5_programming_language_application.pdf
        LM2001020235_23v1_data_structure_use.pdf
      converted_md/
        LM2001020231_23v5_programming_language_use.md
        LM2001020230_23v5_programming_language_application.md
        LM2001020235_23v1_data_structure_use.md
      reports/
        ncs_report_001.xls
      old/
      ncs_application_sw_programming.yaml
    materials/
      tutorial_functions.md
      python_tutorial_data_structures.md
      10min.rst
      employ24_metadata.md
      README_DATASET.md
  processed/
    selected_sources.yaml
    chunks.jsonl
    chunk_index.csv
    dataset_manifest.json
  gold/
    retrieval_gold.jsonl
    generation_gold.yaml
    human_eval_rubric.yaml
```

권장 정리:

- `data/raw/materials/ncs/ncs.yaml`은 `data/raw/ncs/ncs_application_sw_programming.yaml`로 재작성한다.
- `LM*.pdf`와 `LM*.md`는 `source_pdf/`, `converted_md/`로 분리한다.
- 한국어·특수문자 파일명은 사람이 보기에는 좋지만 자동화 스크립트에서 인코딩 문제가 생길 수 있으므로, 전처리 입력용 파일은 ASCII snake_case 별칭으로 복사하거나 manifest에서 alias를 부여한다.
- `report.xls`, `report (1).xls` 형태의 파일은 `ncs_report_001.xls`처럼 순번 기반 이름으로 정리한다.
- 기존 구버전 보고서는 `data/raw/ncs/old/`에 보관하고 전처리 입력에서 제외한다.
- Python/pandas 전체 압축 해제본은 로컬 보관하되, MVP 전처리 입력에서는 제외한다.
- 전처리 스크립트는 `selected_sources.yaml`에 지정된 파일만 읽도록 한다.

## 6. 전처리 파이프라인

### 6.1 입력 선별

1. `selected_sources.yaml`을 읽는다.
2. 존재하지 않는 파일이 있으면 실패 처리한다.
3. `license`, `source_url`, `source_name`이 비어 있으면 실패 처리한다.
4. `pandas-main/`, `python-3.14-docs-html/`, `python-3.14-docs-text/`, `ncs/source_pdf/` 전체 폴더는 기본 입력에서 제외한다.
5. NCS 자료는 PDF가 아니라 `converted_md/`의 선별 MD 파일만 입력으로 사용한다.
6. `old/`와 `reports/`는 기본 입력에서 제외하고, YAML 작성 검증용 참고 자료로만 사용한다.

### 6.2 텍스트 정제

정제 규칙:

- 연속 공백과 빈 줄을 줄인다.
- 페이지 머리말, 목차, navigation 텍스트를 제거한다.
- RST 역할 표기인 `:class:`, `:func:`, `:ref:`는 사람이 읽는 평문으로 변환한다.
- 코드 블록은 보존하되, 앞뒤 설명 문단과 분리되지 않게 한다.
- 한국어/영어 혼합 metadata는 UTF-8로 저장한다.

### 6.3 Chunking

초기 설정:

| 항목 | 값 |
|---|---:|
| chunk 크기 | 800자 |
| overlap | 120자 |
| 최소 chunk 길이 | 250자 |
| 최대 chunk 길이 | 1,200자 |
| 우선 분리 기준 | 제목, 소제목, 빈 줄, 문단 |

품질 규칙:

- 하나의 chunk에는 하나의 핵심 개념만 들어가게 한다.
- 코드 예제만 단독으로 남은 chunk는 지양한다.
- `def`, `parameter`, `return`, `list`, `dict`, `DataFrame` 등 핵심 키워드를 tags로 부여한다.
- citation에 사용할 `chunk_id`는 재실행해도 안정적으로 생성되어야 한다.

### 6.4 NCS PDF/MD 정리

PDF는 이미 MD sidecar로 변환했지만, 파일명과 위치가 아직 전처리 자동화에 최적화되어 있지 않다. 정리 작업은 다음 순서로 수행한다.

1. `data/raw/ncs/source_pdf/`를 생성한다.
2. MVP 후보 PDF 3개를 ASCII alias 파일명으로 복사한다.
3. `data/raw/ncs/converted_md/`를 생성한다.
4. 같은 3개 MD를 ASCII alias 파일명으로 복사한다.
5. 나머지 NCS PDF/MD는 `data/raw/ncs/archive/` 또는 기존 `data/raw/materials/ncs/`에 보관하되, `selected_sources.yaml`에서 제외한다.
6. `report*.xls`는 `data/raw/ncs/reports/ncs_report_001.xls` 형식으로 정리한다.
7. `old/` 폴더의 기존 보고서는 보존하되 기본 전처리 대상에서 제외한다.

권장 alias:

| 원본 | 권장 파일명 |
|---|---|
| `LM2001020231_23v5_.프로그래밍+언어+활용_20251108.pdf` | `LM2001020231_23v5_programming_language_use.pdf` |
| `LM2001020230_23v5_프로그래밍+언어+응용_20251108.pdf` | `LM2001020230_23v5_programming_language_application.pdf` |
| `LM2001020235_23v1_자료구조+활용_20251108.pdf` | `LM2001020235_23v1_data_structure_use.pdf` |

| 데이터 | 목적 | 생성 방식 | 산출 파일 | 우선순위 |
|---|---|---|---|---:|

| Python 함수 실습 예시 3개 | 실습 생성 품질 비교 | 함수 정의, 문자열 변환, list/dict 처리 과제 3개 작성 | `data/raw/synthetic/practice_examples.yaml` | 중간 |
| 실제 사용성 테스트 기록 양식 | 실증 검증 | 2~3명 사용성 테스트를 가정한 빈 기록 양식 작성 | `data/gold/usability_test_form.yaml` | 중간 |
| NCS YAML 검토 체크리스트 | PDF/MD에서 요약한 수행준거의 과잉 해석 방지 | 원문 출처, 요약 근거, 재사용 제한 확인 항목 작성 | `data/gold/ncs_yaml_review_checklist.yaml` | 높음 |
| 파일명 매핑 manifest | 원본 한글 파일명과 ASCII alias 연결 | 원본명, alias, 역할, 채택 여부를 CSV로 작성 | `data/processed/source_file_map.csv` | 높음 |

임의 생성 규칙:

- 모든 합성 데이터에는 `synthetic: true`, `created_for: "LessonPack AI MVP"`, `created_at`을 포함한다.
- 생성형 AI가 작성한 샘플과 사람이 작성한 샘플을 구분할 수 있도록 `authoring_method`를 기록한다.
- 외부 원문처럼 보이는 URL, 기관명, 실제 강사명, 실제 수강생명은 넣지 않는다.
- 커리큘럼과 실습 예시는 Python 공식 문서 chunk와 NCS YAML을 연결할 수 있는 범위로만 작성한다.
- 검증용 gold 데이터는 정답을 과도하게 넓게 잡지 않고, 실제 chunk ID를 기준으로 작성한다.

## 10. 구현 작업 순서

1. `data/raw/curriculum/`, `data/raw/ncs/source_pdf/`, `data/raw/ncs/converted_md/`, `data/raw/ncs/reports/`, `data/raw/ncs/old/` 구조를 만든다.
2. NCS 후보 PDF 3개와 MD 3개를 전처리 입력용 ASCII alias 파일명으로 복사한다.
3. `report*.xls` 파일을 `reports/`로 이동하거나 복사하고 순번 파일명으로 정리한다.
4. 커리큘럼 YAML 1개를 작성한다.
5. 변환 MD를 기준으로 NCS YAML 1개를 작성한다.
6. NCS YAML에 `source_md`, `source_url`, `license_note`, `selected_reason`, `learning_topics`를 포함한다.
7. `data/raw/synthetic/`를 만들고 Python 함수 실습 예시 3개를 합성 작성한다.
8. `data/gold/human_eval_rubric.yaml`, `data/gold/usability_test_form.yaml`, `data/gold/ncs_yaml_review_checklist.yaml`을 임의 생성한다.
9. 원본 파일명과 alias 파일명을 연결하는 `data/processed/source_file_map.csv`를 생성한다.
10. `selected_sources.yaml`을 작성한다.
11. 선택된 원문만 읽는 전처리 스크립트를 만든다.
12. Python/RST/NCS MD 정제 규칙을 적용한다.
13. `chunks.jsonl`과 `chunk_index.csv`를 생성한다.
14. 사람이 `chunk_index.csv`를 검토해 부적합 chunk를 제외한다.
15. retrieval gold query 10개를 작성한다.
16. generation gold case 3개를 작성한다.
17. `dataset_manifest.json`을 생성한다.
18. 검증 프로토콜의 Level 0, Level 3 기준으로 데이터셋을 점검한다.

## 11. 완료 정의

데이터셋 전처리 준비가 완료되었다고 판단하려면 다음을 만족해야 한다.

- `data/raw/curriculum/`에 커리큘럼 YAML 1개가 있다.
- `data/raw/synthetic/practice_examples.yaml`에 Python 함수 실습 예시 3개가 있다.
- `data/raw/ncs/`에 NCS YAML 1개가 있고 0바이트가 아니다.
- NCS 후보 PDF와 MD가 `source_pdf/`, `converted_md/`에 분리되어 있다.
- 전처리 입력 파일명은 ASCII alias 또는 manifest alias로 안정적으로 참조된다.
- `data/processed/chunks.jsonl`에 30개 이상의 chunk가 있다.
- 모든 chunk에 출처 URL, 라이선스, section, chunk ID가 있다.
- `data/gold/retrieval_gold.jsonl`에 10개 이상의 query가 있다.
- `data/gold/generation_gold.yaml`에 3개 이상의 case가 있다.
- `data/gold/human_eval_rubric.yaml`, `data/gold/usability_test_form.yaml`, `data/gold/ncs_yaml_review_checklist.yaml`이 있다.
- 임의 생성 데이터에는 `synthetic: true`와 `authoring_method`가 기록되어 있다.
- `data/processed/dataset_manifest.json`으로 데이터셋 버전과 수량을 확인할 수 있다.
- MVP에서 제외할 원천 자료가 전처리 입력으로 들어가지 않는다.
