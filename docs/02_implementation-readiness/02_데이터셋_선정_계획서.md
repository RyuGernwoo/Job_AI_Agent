# LessonPack AI MVP 데이터셋 선정 계획서

작성일: 2026-07-04
대상 서비스: 직업훈련 강의 운영 보조 AI Agent
목적: RAG 검색, 강의 패키지 생성, 검증을 위한 데이터셋 후보와 선정 기준을 정의한다.

---

## 1. 데이터셋 선정 원칙

| 원칙 | 설명 |
|---|---|
| 저작권 우선 | 외부 API나 LLM에 넣을 자료는 공개 라이선스 또는 사용 허가 자료만 사용 |
| MVP 적합성 | 전체 과정이 아니라 1개 차시 생성에 필요한 최소 데이터만 선정 |
| 근거 추적 가능성 | 모든 원문은 source URL, 라이선스, 수집일, chunk ID를 기록 |
| NCS 정합성 | NCS 능력단위는 자동 크롤링하지 않고 수동 선정·기록 |
| 평가 가능성 | 검색 정답 chunk와 기대 산출물 기준을 별도로 만든다 |

## 3. MVP 데이터셋 구성

MVP에는 다음 5종 데이터를 사용한다.

| ID | 데이터셋 | 필수 여부 | 설명 |
|---|---|---|---|
| D-01 | 커리큘럼 샘플 | 필수 | 1개 과정, 1개 차시, 학습목표, 학습자 수준 |
| D-02 | NCS 능력단위 샘플 | 필수 | 수동 선정한 능력단위명, 수행준거, 지식·기술·태도 |
| D-03 | 교재 원문 샘플 | 필수 | 공개 라이선스 Python/Pandas 문서 일부 또는 사용 허가 교재 |
| D-04 | 훈련과정 metadata | 선택 | 고용24/HRD-Net 유사 과정명과 설명을 시장 맥락 참고용으로만 사용 |
| D-05 | 평가 gold set | 필수 | retrieval query, 정답 chunk ID, 기대 산출물 품질 기준 |

## 4. 권장 MVP 주제 데이터셋

### 4.1 샘플 과정

| 항목 | 값 |
|---|---|
| 과정명 | 생성형 AI 활용 Python 기초 |
| 차시명 | Python 함수와 프롬프트 자동화 실습 |
| 학습 대상 | Python 기초 문법을 학습한 직업훈련 수강생 |
| 선수 지식 | 변수, 조건문, 반복문 |
| 차시 목표 | 함수 정의, 입력값 처리, 간단한 자동화 스크립트 작성 |
| 실습 목표 | 사용자 입력을 받아 텍스트를 변환하는 Python 함수 작성 |

이 주제는 Python 공식 문서와 연결하기 쉽고, 실습·평가 문항 생성이 명확하다.

### 4.2 NCS 데이터

NCS 데이터는 다음 형식으로 수동 작성한다.

```yaml
ncs_unit:
  unit_code: "수동 입력"
  unit_name: "수동 입력"
  source_url: "https://www.ncs.go.kr/"
  selected_reason: "Python 자동화/프로그래밍 직무 능력과 관련"
  performance_criteria:
    - "수행준거 1"
    - "수행준거 2"
  knowledge:
    - "필요 지식"
  skills:
    - "필요 기술"
  attitude:
    - "필요 태도"
  license_note: "공식 사이트 조회 후 교육 목적 샘플로 수동 정리"
```

주의:

- NCS 공식 데이터를 자동 대량 수집하지 않는다.
- MVP에서는 능력단위 1~2개만 수동 선정한다.
- 실제 코드 repo에는 원문 전체가 아니라 필요한 범위의 요약·참조 정보를 저장한다.

### 4.3 교재 원문 데이터

1차 후보:

| 자료 | 사용 범위 | 이유 |
|---|---|---|
| Python Tutorial | 함수, 자료구조, 입출력 관련 일부 섹션 | Python 기초 수업에 바로 연결 가능 |
| pandas User Guide | DataFrame 기초 일부 | 데이터 분석 직업훈련 차시로 확장 가능 |
| 사용자 작성 교안 | 프로젝트 맞춤형 교재 | 저작권 리스크 최소화 |

Python 공식 문서는 페이지 하단에 PSF License와 문서 코드 예제의 Zero-Clause BSD 라이선스 안내가 있다. 사용 시 source URL과 라이선스 메모를 반드시 남긴다.

| 기준 | 배점 | 설명 |

## 8. 데이터 수집 절차

1. 수집 후보 URL과 라이선스 확인
2. `README_DATASET.md`에 출처, 접근일, 라이선스 기록
3. 원문 일부를 `data/raw/materials/`에 저장
4. 수동 커리큘럼과 NCS 요약 작성
5. chunking 실행
6. chunk ID와 source metadata 검증
7. retrieval gold set 작성
8. generation gold set 작성
9. 검증 프로토콜에서 사용

## 10. 데이터셋 완료 정의

아래 조건을 만족하면 MVP 데이터셋 준비 완료로 본다.

- 커리큘럼 YAML 1개가 있다.
- NCS 능력단위 YAML 1개 이상이 있다.
- 교재 원문 chunk 30개 이상이 있다.
- 모든 chunk에 source URL, license, section metadata가 있다.
- retrieval gold query 10개 이상이 있다.
- generation gold case 3개 이상이 있다.
- 저작권 불명확 자료가 없다.
