## 0. 현재 구현 반영 요약

2026-07-21 기준 MVP 구현은 1차 완료 상태다. 현재 코드는 FastAPI API, Streamlit UI, PyMuPDF 기반 PDF 추출, Supabase vector store, LiteLLM provider, OpenAI primary/Gemini fallback, Langfuse tracing, DOCX/PPTX export, Docker/GCE CI/CD를 포함한다. 두 차례의 실제 산출물 분석을 거쳐 citation 상세 출처, NCS alignment, 검수 이력, 본문 수정 API, 구조화 LLM JSON 검증, 한국어 export label, 마지막 출처 단원 통합, 수업 제목 기반 파일명, export 품질 검사 스크립트를 반영했다.

이 문서의 “구현 순서”와 일부 후보 비교는 초기 설계 맥락을 보존한다. 실제 실행 기준은 `README.md`, `.env.example`, `config.example.yaml`, `supabase/migrations/001_lessonpack_vectors.sql`과 현재 `src/` 코드를 우선한다.

## 1. 목적

이 문서는 LessonPack AI MVP를 실제 구현하기 위한 기술 명세서다.
MVP의 목표는 다음 end-to-end 흐름을 한 달 안에 구현·시연하는 것이다.

```text
커리큘럼/NCS 입력
→ 교재 업로드
→ 문서 파싱·chunking
→ RAG 근거 검색
→ 교안·실습·평가 생성
→ 강사 검수·수정
→ DOCX/PPTX 다운로드
→ 생성 로그 저장
```

MVP는 전체 LMS가 아니다. 1개 차시의 강의 패키지를 생성·검수·다운로드하는 데 집중한다.

## 3. 구현 범위

### 3.1 Must Have

| ID | 기능 | 구현 기준 |
|---|---|---|
| M-01 | 프로젝트 생성 | 과정명, 차시명, 학습목표, NCS 능력단위 입력 |
| M-02 | 교재 업로드 | PDF/TXT 우선 지원, DOCX는 후순위 |
| M-03 | 문서 chunking | chunk ID, 원문, 문서명, 페이지/위치 metadata 저장 |
| M-04 | RAG 검색 | 차시명·학습목표·NCS를 query로 top-k chunk 반환 |
| M-05 | 강의 패키지 생성 | 교안, 실습, 객관식 5문항, 실습형 평가 1개 생성 |
| M-06 | citation 연결 | 생성 항목별 `citation_ids`와 `evidence_sources` 필수 |
| M-07 | 강사 검수 | draft → reviewed → approved 상태 전환, reviewer notes와 review history 저장 |
| M-08 | DOCX/PPTX export | approved 패키지에서 한국어 label, NCS alignment, 검수 이력과 출처별로 묶은 마지막 근거 단원이 포함된 DOCX/PPTX 생성. 다운로드명은 `{수업명}_교안.docx`, `{수업명}_강의자료.pptx` 형식 |
| M-09 | 로그 저장 | 입력, 검색 chunk, 프롬프트, 생성 결과, 수정 이력 저장 |

### 3.2 Should Have

- 생성 결과 diff 보기
- RAGAS 기반 평가 확장

### 3.3 Won't Have

- LMS 연동
- SCORM/xAPI export
- 자동 채점 운영
- 기관 계정/권한 관리

### 4.2 LLM provider adapter

LLM 공급자는 구현 코드에 직접 고정하지 않는다. 다음 interface로 감싼다.

```python
class LLMProvider:
    def generate_json(self, prompt: str, schema_name: str) -> dict:
        ...
```

초기 구현에서는 아래 중 하나만 연결한다.

| 후보 | 장점 | 리스크 |
|---|---|---|
| CLOVA Studio / HyperCLOVA X | 한국어 맥락에 강점 | API 접근 권한과 비용 확인 필요 |
| OpenAI/Gemini API | LiteLLM으로 primary/fallback 라우팅 가능 | API quota, 비용, 교재 외부 전송 정책 확인 필요 |
| 로컬 LLM | 데이터 외부 전송 최소화 | 품질과 속도 리스크 |

명확하지 않은 provider는 코드에 숨겨 넣지 않는다. `.env` 또는 `config.yaml`에 명시한다.

## 6. 명시적 설정값

기재되지 않은 파라미터는 기본값으로 암묵 제공하지 않는다. 모든 실행 파라미터는 `config.yaml`에 명시한다.

| 설정명 | 초기값 | 설명 |
|---|---:|---|
| `chunk_size_chars` | 800 | 한국어/영문 혼합 문서 기준 1개 chunk 문자 수 |
| `chunk_overlap_chars` | 120 | chunk 경계 손실 방지용 overlap |
| `retrieval_top_k` | 5 | 생성 프롬프트에 넣을 검색 chunk 수 |
| `max_upload_mb` | 20 | MVP 업로드 파일 크기 제한 |
| `allowed_file_types` | `["pdf", "txt", "md"]` | MVP 지원 파일 |
| `llm.provider` | 명시 필요 | `mock`, `http_chat`, `litellm` 중 선택 |
| `llm.model` | 명시 필요 | mock 또는 외부 chat model 이름 |
| `llm.base_url` | provider별 명시 | `http_chat` provider endpoint |
| `llm.api_key_env` | provider별 명시 | API key를 읽을 환경변수명 |
| `llm.timeout_seconds` | provider별 명시 | 외부 provider 요청 timeout |
| `embedding_provider` | 명시 필요 | local 또는 API provider 선택 |
| `export_docx_enabled` | true | DOCX export 사용 |
| `export_pptx_enabled` | true | PPTX 요약 export 사용 |
| `citation_required` | true | citation 없는 핵심 항목 실패 처리 |
| `vector_store.provider` | `memory` 또는 `supabase` | 로컬 테스트는 memory, 운영 실증은 Supabase(pgvector) |
| `vector_store.table_name` | `lessonpack_chunks` | Supabase chunk 저장 table |
| `vector_store.match_function` | `match_lessonpack_chunks` | Supabase RPC 검색 함수 |
| `SUPABASE_URL` | 환경변수 | Supabase project URL |
| `SUPABASE_SERVICE_ROLE_KEY` | 환경변수 | 서버 전용 Supabase key, Git 커밋 금지 |

## 11. Export 명세

### 11.1 DOCX

DOCX는 MVP 필수 산출물이다.

섹션:

1. 표지
2. 과정·차시 정보
3. NCS 능력단위
4. 학습목표
5. 교안
6. 실습 시나리오
7. 평가 문항
8. 실습형 평가 루브릭
9. 검수 이력
10. 근거 출처 목록: 본문에서 사용한 근거만 원천별로 묶어 URL·라이선스·청크 ID 표시

### 11.2 PPTX

PPTX는 발표와 실증 설명을 위한 요약 산출물이다. 승인된 패키지에서만 생성하며, citation ID는 본문 슬라이드에 반복하지 않고 마지막 근거 출처 슬라이드에만 표시한다.

슬라이드:

1. 차시 개요
2. 학습목표
3. 핵심 개념
4. 실습 안내
5. 평가 안내
6. 검수 이력
7. 근거 출처: 원천별 그룹, 사용 청크, 라이선스

## 12. 오류 처리

| 오류 | 처리 |
|---|---|
| 지원하지 않는 파일 | 400, 허용 확장자 반환 |
| 파일 크기 초과 | 413, `max_upload_mb` 안내 |
| 텍스트 추출 실패 | 422, 수동 TXT 업로드 안내 |
| 검색 결과 없음 | 200 + `retrieved_chunks=[]`, 생성 버튼 비활성화 |
| LLM JSON 파싱 실패 | 재시도 1회 후 실패 로그 저장 |
| citation 누락 | draft 저장 금지 또는 `needs_revision` 상태 |
| approved 이전 export | 409 |

## 14. 구현 완료 정의

아래 조건을 모두 만족하면 MVP 구현 완료로 본다.

- 샘플 데이터셋 1개로 파일 업로드가 가능하다.
- 검색 결과 top-5 chunk가 반환된다.
- 교안·실습·평가 JSON이 Pydantic 검증을 통과한다.
- 핵심 생성 항목 90% 이상에 citation ID가 있다.
- 강사 검수 상태가 `approved`로 저장된다.
- approved 패키지에서 DOCX/PPTX가 생성된다.
- 입력, 검색 chunk, 프롬프트, 결과, 수정 이력이 로그로 남는다.
