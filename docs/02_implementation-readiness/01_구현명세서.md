## 1. 목적

이 문서는 LessonPack AI MVP를 실제 구현하기 위한 기술 명세서다.
MVP의 목표는 다음 end-to-end 흐름을 한 달 안에 구현·시연하는 것이다.

```text
커리큘럼/NCS 입력
→ 교재 업로드
→ 문서 파싱·chunking
→ RAG 근거 검색
→ 교안·실습·평가 생성
→ 강사 검수·수정
→ DOCX 다운로드
→ 생성 로그 저장
```

MVP는 전체 LMS가 아니다. 1개 차시의 강의 패키지를 생성·검수·다운로드하는 데 집중한다.

## 2. Deep Research 기반 기술 선택

| 영역 | 후보 | 채택 | 근거 |
|---|---|---|---|
| API 서버 | FastAPI, Flask, Django | **FastAPI** | 파일 업로드, Pydantic 기반 schema, OpenAPI 문서화에 적합. FastAPI 공식 문서는 `UploadFile`이 큰 파일 처리에 유리하다고 설명한다. |
| UI | Streamlit, React, 단순 HTML | **Streamlit 우선** | 개인 MVP에서 빠른 데모에 유리. 필요 시 FastAPI 단독 API 문서로도 시연 가능 |
| Vector DB | Chroma, FAISS, Qdrant | **Chroma** | 로컬 실행이 쉽고 collection에 document, id, metadata를 저장·검색할 수 있다. |
| 문서 파싱 | pypdf, PyMuPDF, unstructured | **pypdf 우선** | PDF 텍스트 추출 MVP에 충분. 복잡한 PDF는 추후 PyMuPDF 또는 unstructured 검토 |
| 구조화 검증 | 직접 dict 검증, Pydantic | **Pydantic** | 생성 결과 JSON schema 검증과 API DTO 일관성 유지에 적합 |
| DOCX export | python-docx | **python-docx** | 교안·실습·평가 문서 생성에 적합 |
| PPTX export | python-pptx | **Should Have** | 발표 요약 슬라이드 생성 가능하나 DOCX 안정화 후 구현 |
| HITL | 상태값 직접 관리, LangGraph interrupt | **상태값 직접 관리 우선** | MVP에서는 `draft/reviewed/approved` 상태로 충분. LangGraph interrupt는 2차 확장 후보 |
| 평가 | 수동 루브릭, RAGAS, 자체 지표 | **혼합** | RAG 검색은 context precision/recall류 지표, 산출물 품질은 사람 루브릭으로 평가 |

## 3. 구현 범위

### 3.1 Must Have

| ID | 기능 | 구현 기준 |
|---|---|---|
| M-01 | 프로젝트 생성 | 과정명, 차시명, 학습목표, NCS 능력단위 입력 |
| M-02 | 교재 업로드 | PDF/TXT 우선 지원, DOCX는 후순위 |
| M-03 | 문서 chunking | chunk ID, 원문, 문서명, 페이지/위치 metadata 저장 |
| M-04 | RAG 검색 | 차시명·학습목표·NCS를 query로 top-k chunk 반환 |
| M-05 | 강의 패키지 생성 | 교안, 실습, 객관식 5문항, 실습형 평가 1개 생성 |
| M-06 | citation 연결 | 생성 항목별 `citation_ids` 필수 |
| M-07 | 강사 검수 | draft → reviewed → approved 상태 전환 |
| M-08 | DOCX export | approved 패키지에서 DOCX 생성 |
| M-09 | 로그 저장 | 입력, 검색 chunk, 프롬프트, 생성 결과, 수정 이력 저장 |

### 3.2 Should Have

- PPTX 요약 export
- Streamlit UI
- 생성 결과 diff 보기
- RAGAS 또는 자체 평가 스크립트

### 3.3 Won't Have

- LMS 연동
- SCORM/xAPI export
- 자동 채점 운영
- 기관 계정/권한 관리
- NCS 공식 DB 자동 크롤링
- 전체 과정 일괄 생성

|---|---|

| Export | python-docx, 선택적으로 python-pptx |
| LLM | provider adapter 방식으로 분리 |

### 4.2 LLM provider adapter

LLM 공급자는 구현 코드에 직접 고정하지 않는다. 다음 interface로 감싼다.

```python
class LLMProvider:
    def generate_json(self, prompt: str, schema_name: str) -> dict:
        ...
```

초기 구현에서는 아래 중 하나만 연결한다.

| 후보 | 장점 | 리스크 |
|---|---|---|
| CLOVA Studio / HyperCLOVA X | 한국어 맥락에 강점 | API 접근 권한과 비용 확인 필요 |
| OpenAI-compatible API | 자료와 예제가 많음 | 교재 외부 전송 정책 확인 필요 |
| 로컬 LLM | 데이터 외부 전송 최소화 | 품질과 속도 리스크 |

명확하지 않은 provider는 코드에 숨겨 넣지 않는다. `.env` 또는 `config.yaml`에 명시한다.

## 6. 명시적 설정값

기재되지 않은 파라미터는 기본값으로 암묵 제공하지 않는다. 모든 실행 파라미터는 `config.yaml`에 명시한다.

| 설정명 | 초기값 | 설명 |
|---|---:|---|
| `chunk_size_chars` | 800 | 한국어/영문 혼합 문서 기준 1개 chunk 문자 수 |
| `chunk_overlap_chars` | 120 | chunk 경계 손실 방지용 overlap |
| `retrieval_top_k` | 5 | 생성 프롬프트에 넣을 검색 chunk 수 |
| `max_upload_mb` | 20 | MVP 업로드 파일 크기 제한 |
| `allowed_file_types` | `["pdf", "txt", "md"]` | MVP 지원 파일 |
| `llm_provider` | 명시 필요 | `mock`, `openai_compatible`, `clova`, `local` 중 선택 |
| `embedding_provider` | 명시 필요 | local 또는 API provider 선택 |
| `export_docx_enabled` | true | DOCX export 사용 |
| `export_pptx_enabled` | false | PPTX export는 후순위 |
| `citation_required` | true | citation 없는 핵심 항목 실패 처리 |

  "chunk_id": "doc001-p001-c003",

### 7.3 LessonPackage

```json
{
  "package_id": "uuid",
  "project_id": "uuid",
  "status": "draft",
  "lesson_plan": {
    "title": "차시명",
    "learning_objectives": ["목표 1"],
    "lecture_flow": [
      {
        "section": "도입",
        "duration_min": null,
        "content": "강의 내용",
        "citation_ids": ["doc001-p001-c003"]
      }
    ]
  },
  "practice": {
    "scenario": "실습 상황",
    "steps": ["1단계", "2단계", "3단계"],
    "submission": "제출물",
    "rubric": ["평가 기준 1"],
    "citation_ids": ["doc001-p002-c001"]
  },
  "assessment": {
    "multiple_choice": [],
    "performance_task": {}
  }
}
```

## 8. API 명세

| Method | Endpoint | 입력 | 출력 |
|---|---|---|---|
| POST | `/api/projects` | ProjectCreate | Project |
| POST | `/api/projects/{project_id}/materials` | multipart file | MaterialIngestResult |
| POST | `/api/projects/{project_id}/retrieve` | RetrieveRequest | RetrievedChunk[] |
| POST | `/api/projects/{project_id}/generate` | GenerateRequest | LessonPackage |
| GET | `/api/packages/{package_id}` | path param | LessonPackage |
| PATCH | `/api/packages/{package_id}/review` | ReviewPatch | LessonPackage |
| GET | `/api/packages/{package_id}/export.docx` | path param | file |
| GET | `/api/packages/{package_id}/generation-log` | path param | GenerationLog |

### 8.1 상태 전이

```text
draft → reviewed → approved → exported
draft → regenerated
reviewed → draft
approved → exported
```

`approved` 이전에는 export를 막는다.

## 10. 생성 프롬프트 원칙

| 원칙 | 내용 |
|---|---|
| 근거 제한 | 제공된 retrieved chunks 밖의 사실을 단정하지 않는다. |
| 구조 고정 | JSON schema를 반드시 따른다. |
| citation 필수 | 핵심 설명, 실습, 평가 문항에 citation ID를 붙인다. |
| 불확실성 표시 | 근거가 부족하면 `근거 부족`으로 표시한다. |
| NCS 연결 | 생성 항목이 어떤 NCS 능력단위와 연결되는지 명시한다. |

## 12. 오류 처리

| 오류 | 처리 |
|---|---|
| 지원하지 않는 파일 | 400, 허용 확장자 반환 |
| 파일 크기 초과 | 413, `max_upload_mb` 안내 |
| 텍스트 추출 실패 | 422, 수동 TXT 업로드 안내 |
| 검색 결과 없음 | 200 + `retrieved_chunks=[]`, 생성 버튼 비활성화 |
| LLM JSON 파싱 실패 | 재시도 1회 후 실패 로그 저장 |
| citation 누락 | draft 저장 금지 또는 `needs_revision` 상태 |
| approved 이전 export | 409 |

## 14. 구현 완료 정의

아래 조건을 모두 만족하면 MVP 구현 완료로 본다.

- 샘플 데이터셋 1개로 파일 업로드가 가능하다.
- 검색 결과 top-5 chunk가 반환된다.
- 교안·실습·평가 JSON이 Pydantic 검증을 통과한다.
- 핵심 생성 항목 90% 이상에 citation ID가 있다.
- 강사 검수 상태가 `approved`로 저장된다.
- approved 패키지에서 DOCX가 생성된다.
- 입력, 검색 chunk, 프롬프트, 결과, 수정 이력이 로그로 남는다.
